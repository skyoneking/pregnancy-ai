## ADDED Requirements

### Requirement: 知识库数据结构
系统 SHALL 定义结构化的知识库数据，支持按阶段、周数、关键词索引。

#### Scenario: 备孕期知识结构
- **WHEN** 查询备孕期知识
- **THEN** 知识库 MUST 包含以下知识点：
  - 叶酸补充（补充原因、时间、剂量）
  - 孕前检查（项目清单、时间）
  - 排卵期计算（计算方法、受孕窗口）
  - 生活调整（戒烟酒、作息、运动）
  - 营养准备（叶酸、铁、碘、蛋白质）
  - 情绪管理（减压技巧、夫妻沟通）
- **AND** 每个知识点包含：`id`、`title`、`content`、`stage`、`tags`

#### Scenario: 孕期知识结构
- **WHEN** 查询孕期知识
- **THEN** 知识库 MUST 包含每周发育信息（1-42 周）
- **AND** 每周信息包含：胎儿发育、母体变化、注意事项
- **AND** 知识库 MUST 包含营养指导（每周重点营养素）
- **AND** 知识库 MUST 包含运动建议（适合各孕周的运动）
- **AND** 知识库 MUST 包含心理变化（常见情绪波动及应对）

#### Scenario: 产后期知识结构
- **WHEN** 查询产后期知识
- **THEN** 知识库 MUST 包含以下知识点：
  - 产后 42 天恢复（恶露、伤口、复查）
  - 母乳喂养（含接姿势、追奶、离乳）
  - 新生儿护理（换尿布、洗澡、睡眠）
  - 宝宝发育（大运动、精细动作、语言）
  - 疫苗接种（时间表、注意事项）
  - 妈妈恢复（盆底肌、腹直肌、情绪）
- **AND** 每个知识点包含：`id`、`title`、`content`、`stage`、`relevant_days`

### Requirement: 知识内容格式
系统 SHALL 使用统一的 Markdown 格式存储知识内容。

#### Scenario: 标准知识条目格式
- **WHEN** 定义知识条目
- **THEN** 内容格式 MUST 包含：
  - 标题（简洁明确）
  - 正文（100-200 字，通俗易懂）
  - 关键标签（用于检索）
  - 免责声明（"请以医生意见为准"）
- **AND** 内容使用中文编写
- **AND** 医学术语提供通俗解释

#### Scenario: 孕周特定内容格式
- **WHEN** 定义孕周特定知识
- **THEN** 内容格式 MUST 包含：
  - 孕周标识（如"孕 20 周"）
  - 胎儿发育状态
  - 本周重要事项（如产检、营养重点）
  - 常见问题解答

### Requirement: 知识库索引机制
系统 SHALL 提供按多个维度快速检索知识的能力。

#### Scenario: 按阶段索引
- **WHEN** 查询指定阶段的知识
- **THEN** 系统返回该阶段的所有相关知识
- **AND** 结果按相关性排序

#### Scenario: 按孕周索引
- **WHEN** 查询指定孕周的知识（孕期）
- **THEN** 系统返回该孕周的发育信息和注意事项
- **AND** 如果该周无专门内容，返回最接近的孕周内容

#### Scenario: 按关键词索引
- **WHEN** 查询包含特定关键词的知识
- **THEN** 系统返回标题或内容中包含该关键词的所有知识
- **AND** 结果按匹配度排序

#### Scenario: 按产后天数索引
- **WHEN** 查询指定产后天数的知识（产后）
- **THEN** 系统返回该天数相关的恢复和护理知识
- **AND** 优先返回最接近天数的内容

### Requirement: 知识库查询 API
系统 SHALL 提供内部 API 供 AI Agent 调用知识库。

#### Scenario: Agent 查询阶段知识
- **WHEN** AI Agent 调用知识库查询指定阶段
- **THEN** API 返回该阶段的核心知识点
- **AND** 返回格式为结构化 JSON（易于 AI 处理）

#### Scenario: Agent 查询孕周知识
- **WHEN** AI Agent 调用知识库查询指定孕周
- **THEN** API 返回该孕周的发育信息和本周重点
- **AND** 如果无该周专门内容，返回最接近周并注明

#### Scenario: Agent 按关键词查询
- **WHEN** AI Agent 调用知识库查询特定关键词
- **THEN** API 返回匹配的知识条目
- **AND** 返回最多 3 条最相关内容

### Requirement: 知识内容准确性保障
系统 SHALL 在所有知识内容中包含医学免责声明。

#### Scenario: 所有知识条目标注免责声明
- **WHEN** AI 返回知识库内容给用户
- **THEN** 内容末尾 MUST 包含"以上仅供参考，不构成医疗诊断，请以医生意见为准"
- **AND** 医疗建议相关内容 MUST 特别强调咨询专业医生

#### Scenario: 紧急症状优先级提示
- **WHEN** 知识内容涉及紧急症状（如出血、剧烈腹痛）
- **THEN** 内容 MUST 明确标注紧急程度和就医建议
- **AND** 危急症状 MUST 建议"立即就医或拨打 120"
